This update to the Control4 Platform - OS 3.4.2 - is here to elevate your Control4 experience! It's the next milestone in our commitment to provide new products and features to enhance your customer’s lives, while also investing in our Partner Tools to make deployments faster and easier.

We've been hard at work resolving reported issues from previous releases to ensure systems on OS 3.4.2 run smoother and more reliably than ever. This release also includes backend and proxy changes that will set the stage for exciting new additions coming this year.

Stay tuned for all the latest from Control4 and Snap One.

OS 3.3.2:
LIGHTS! CAMERAS!
AND HALO PRE-ORDER ACTION!

Announced at CEDIA with the previous release, Halo and Halo Touch officially launch with the OS 3.3.2 release. This update adds the fully featured Voice Coordinator Agent, and the initial-release drivers for Halo and Halo Touch, which include:

  • Deep Sleep Timeout period to extend battery life (Halo and Halo Touch)
  • Digital Labels and programming for the Custom Buttons (Halo only)

In conjunction with the launch of the new line of Luma x20 IP Cameras, OS 3.3.2 provides new Luma features for the Control4 user interface. Using a Control4 Touchscreen or the Control4 App (for iPhone or iPad, Android phone or tablet) Customers can try out the features and controls in the new Cameras experience. They can view real-time action across multiple cameras simultaneously, rearrange the order of the cameras to fit their needs, and configure the IP Camera settings from the UI itself.

Coming later in Q1, the new Daylight feature will warm and cool the color temperature of tunable white LEDs (CCT) lights, timed to the rising and setting of the sun for improved circadian health. Using the Daylight Agent in Composer Pro, or the new Daylight Tab on the Lights Page, Daylight programs can be created or modified, with options to warm or cool, and brighten or dim the lights. Daylight can be configured to match the Customer’s daily sunrise, midday and sunset times, or Daylight can be configured based on specific times throughout the day, with the option to add additional points for a more controlled and customized experience.

    • IMPORTANT NOTE: This agent will not available on the OS 3.3.2 Launch Day. Daylight will be released through the Online Database later in Q1 2023
    • Automatically adjusts the brightness and color temperatures for tunable white LEDs (CCT) based on the Customer’s local sunrise/sunset times, or custom-set times throughout the day
    • Daylight Programs can be created, modified and removed through the Daylight Agent in Composer Pro by the Partner, or the new Daylight Tab on the Lights Page by Customers
    • Configuration includes selection of the lights to be automated, and increasing/decreasing the brightness and/or color temperature, and extending or reducing the daily schedule to suit the Customers’ preferences
    • Lights must be CCT-capable (tunable white) to participate in Daylight
    • Lights can only participate in a single Daylight Program
    • Changes to color temperature and brightness are imperceptible to the eye, adjusting every 15 minutes
    • If the color temperature range exceeds the capabilities of a light, the light will go to the highest/lowest possible temperature it can deliver
    • Turning the light on/off or changing its color temperature (from a dimmer/switch or a lighting scene) will remove that light from the program for the remainder of the day
    • Turning a Daylight program off and back on will re-enroll any lights that had been removed

Before you update to OS 3

OS 3 is new and different from previous versions of the Control4 OS. Make sure your customers have seen it and tried it before you update their system. Plus, some legacy hardware and software is not supported with OS 3—when updating your current customers to OS 3, consider the following:

  1. Review the OS 3 materials on the Control4 Dealer Portal.
  2. Install OS 3 at your home and showroom, and then demonstrate it to your customers before updating their system. Take the opportunity to show them new features, expand their system, and upgrade outdated hardware.
  3. Current Control4 systems must be running OS 2.9.0 or newer to update to OS 3.

  4. During the update process, Composer will block upgrades to OS 3 if the following products are included in the project.
    Replacing these products is a great upsell opportunity:

    • Controllers (1): HC-250
    • Touch Screens (5): 7" In-Wall v2, 7" Portable v2, 5" In-Wall (FKA: Infinity Edge), 7" In-Wall (FKA: Infinity
    • Edge), 7" In-Wall RevB (FKA: Infinity Edge)
    • Door Stations (2): Exterior Door Station, Interior Door Station
    • AV (1): Speaker Point
    • IP cameras that utilize the Camera Proxy v1
    • WakeUp agent (can be replaced with the Wakeup/Goodnight agent)
    • Security systems that utilize V1 Security drivers
    • Blinds that utilize the Blinds v2 proxy

  5. Graphical UI for the following products is limited or discontinued:

    • iPod proxy: Transport controls (play, skip, pause, etc.) are no longer supported in graphical user interfaces for drivers that use the iPod proxy.
    • The HC-800 is supported in OS 3; however, the Flash-based interface is discontinued, so there is no on-screen display (OSD) for the HC-800. Within Composer, the Upgrade Advisor report will indicate that the HC-800 is supported, but it will not include the OSD.

  6. These are items that are going to be either removed automatically from a project (if they exist) or no longer built and included as a separate product. These are non-impactful items and will be documented only in the Release Notes.

    • Web Nav Access (2) – WebHTML and WebNav (flashnav)
    • Apple Watch
    • MyHome PC
    • Legacy audio
    • EADev Driver
    • Generic media
    • MyHome drivers – control4_ipad, control4_iphone, control4_android, and control4_myhomepc – are removed automatically
    • HC generic driver
    • 4Store agent

Where can I download Composer Pro versions that are older than 2.7.2?

CONTROL4 OS and Composer Pro 2.7.2: T3 7” Tabletop Launch, and Intercom Broadcast Calls

Control4 OS 2.10.6, 2.9.1 and 2.7.2 represent the latest version for each generation of legacy OS releases, representing the most complete offering of available drivers, features and options as well as the most security fixes and bug fixes for each generation of hardware. Control4 highly recommends that Partners update their customers to the highest possible of the above-listed OS versions that is supported by the project’s hardware, at the soonest possible opportunity. Control4 does not recommend any iterative (in-between) build from any OS generation for any customer.

OS 2.7.1 is the oldest OS version that supports the Control4 App for OS 2 for iOS and Android, and the oldest version to support remote access (via Composer Pro, Web Navigator or the Control4 App) due to security enhancements that have been made to the Control4 servers, and any Control4 OS version older than 2.7.1 is no longer supported. Any Partner that still has customers on an OS generation prior to 2.7.1, Control4 recommends that you:

  • Replace any hardware that would prevent an update
  • Download Composer Pro 2.7.2
  • Update your customer to OS 2.7.2

Please keep in mind that OS 2.9.1 (if available based on the hardware in the project) is recommended over OS 2.7.2. Likewise, OS 2.10.6 is preferred over OS 2.9.1. Control4 OS 3 represents the latest in functionality, integration, personalization and reliability. As there have been no end-of-service announcements to date for OS 3, any customer on OS 3.x should be able to update to the latest version: OS 3.3.0.

For any Partner that cannot possibly update a customer and must have an iterative version of Composer Pro in order to properly service or maintain their project, please visit this article on the Tech Community where you can find and download the older legacy iterative versions of Composer Pro.
